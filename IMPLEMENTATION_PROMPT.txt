BACKGROUND (read before writing any code)
==========================================================

BIOLOGY:
- Circular RNAs (circRNAs) in plasma sponge miRNAs via MREs (miRNA Response Elements).
- This "ceRNA network" has stable topology in healthy individuals.
- In early-stage cancer, circRNA hubs are lost or gained → network topology collapses.
- We detect this topological collapse rather than individual DE (differential expression).
  That is the NOVEL CONTRIBUTION over all existing circRNA biomarker papers.
- Datasets: GSE113486 (CRC plasma circRNA, n=188), GSE115513 (miRNA CRC stages, n=344),
  GSE73002 (multi-cancer miRNA, n=1420), GSE159835 (lung validation, n=120).
  All from NCBI GEO — free, no access application needed.

CORE NOVEL CLAIM:
  Healthy ceRNA graphs are modular, hub-dominated, low-entropy networks.
  Stage-I cancer graphs are fragmented, low-modularity, high-entropy.
  XGBoost on 30 graph topology features achieves AUC 0.88–0.93 vs ~0.78 for DE alone.

Skip any file that already exists (cache-safe re-running).

TASK 1B — load_geo.py:
Use GEOparse library to download and parse:
  - GSE113486: plasma circRNA, 188 samples (100 CRC + 88 healthy)
  - GSE115513: plasma miRNA, 344 samples (CRC stages I-IV + healthy)
  - GSE73002:  plasma miRNA, 1420 samples (7 cancer types + healthy)

For each GSE:
  → Try to load from gsm.table (GEOparse standard approach)
  → If tables are empty (common!): scan data/raw/<GSEID>/ for supplementary matrix files

Preprocessing:
  → circRNA: CPM normalisation (counts / (col_sum / 1e6))
  → miRNA:   RPM normalisation (same formula, different name by convention)
  → Filter circRNAs: keep if CPM >= 1.0 in >= 30% of samples
  → Filter miRNAs: keep if CPM >= 1.0 in >= 20% of samples

Metadata parsing from characteristics_ch1:
  → Extract: disease state, disease stage, age, sex
  → Map cancer labels: "colorectal"/"colon"/"rectal" → "CRC", "lung" → "Lung", etc.
  → Map healthy labels: "healthy"/"normal"/"control" → "Healthy"

OUTPUTS:

  → data/processed/miRNA_counts.csv    (features × samples, RPM-normalised, filtered)
  → data/processed/metadata.csv        (sample_id, cancer_type, stage, age, sex, dataset)

PHASE 2: ceRNA NETWORK CONSTRUCTION (network/build_cerna.py)
==========================================================

STEP A — Load interactions:
  → Load circinteractome_interactions.csv: circRNA_id, miRNA_id, edge_weight
  → Load mirtarbase_filtered.csv: miRNA_id (hsa- only), mRNA_id, edge_weight=1.0
  → If CircInteractome has 0 rows: log warning that network will use miRTarBase only

STEP B — Build global tripartite DiGraph (NetworkX DiGraph):
  → circRNA nodes: node_type="circRNA"
  → miRNA nodes:   node_type="miRNA"
  → mRNA nodes:    node_type="mRNA"
  → Edge circRNA→miRNA: weight=normalised_score, edge_type="sponge"
  → Edge miRNA→mRNA:    weight=1.0,             edge_type="suppression"
  → Log: total nodes, edges, breakdown by node_type

STEP C — Per-patient personalised subgraph:
  For each patient ID that exists in circRNA_counts.csv columns:
    a) Get patient's circRNA expression vector (CPM values, index=circRNA names)
    b) Get patient's miRNA expression vector (RPM values) — or population mean if missing
    c) expressed_circ = circRNAs where CPM >= 1.0
    d) expressed_mirna = miRNAs where RPM >= 1.0
    e) keep_nodes = expressed_circ ∪ expressed_mirna ∪ all mRNA nodes (mRNAs always kept)
    f) sub = global_G.subgraph(keep_nodes).copy()
    g) Re-weight each edge: new_weight = sqrt(expr_nodeA * expr_nodeB + 1e-9)
    h) Store: sub.graph["patient_id"] = patient_id
    i) Skip graphs with < 5 nodes (log warning)

OUTPUT: Save dict {patient_id → nx.DiGraph} as network/graphs.pkl using pickle.HIGHEST_PROTOCOL

PHASE 3: TOPOLOGY FEATURE EXTRACTION (features/extract_topology.py)
==========================================================

For each patient graph, extract EXACTLY these 30 features:

GROUP 1 — COMMUNITY STRUCTURE (2 features):
  community_count     : number of Louvain communities (on undirected projection)
                        Use python-louvain if available, else networkx greedy_modularity_communities
  modularity_Q        : Louvain modularity score Q ∈ [-1, 1]

GROUP 2 — HUB ANALYSIS (4 features):
  top5_hub_degree_mean    : mean degree centrality of top-5 circRNA nodes by degree
  top5_hub_degree_max     : max degree centrality among top-5 circRNA nodes
  top5_hub_betweenness_mean : mean betweenness centrality of top-5 circRNA nodes
  top5_hub_betweenness_max  : max betweenness centrality
  NOTE: For graphs > 500 nodes, use k=200 approximation for betweenness

GROUP 3 — CONNECTIVITY (3 features):
  avg_clustering      : networkx.average_clustering on undirected graph
  network_diameter    : diameter of largest connected component (approx for large graphs)
  edge_density        : 2*edges / (nodes*(nodes-1))

GROUP 4 — INFORMATION FLOW (2 features):
  spectral_entropy    : Shannon entropy of normalised abs(eigenvalues) of adjacency matrix
  graph_entropy       : Shannon entropy of node degree distribution

GROUP 5 — RESILIENCE (2 features):
  percolation_threshold : fraction of nodes removed (highest-degree first) when LCC < 50%
  lcc_fragility_score   : area under LCC-vs-removal curve (trapz, normalised)

GROUP 6 — TDA (3 features):
  betti_0             : number of connected components
  betti_1             : approximate cycle count: max(0, edges - nodes + components)
  persistence_entropy : gudhi SimplexTree persistence entropy
                        If gudhi unavailable: scipy entropy of degree distribution as fallback

GROUP 7 — EXPRESSION SUMMARIES (6 features):
  mean_top50_circ_weight  : mean total out-edge weight for top-50 circRNA nodes
  mirna_diversity_shannon : Shannon entropy of miRNA degree distribution
  n_circ_expressed        : number of circRNA nodes in patient's subgraph
  n_mirna_expressed       : number of miRNA nodes
  n_mrna_reachable        : number of mRNA nodes
  circ_mirna_ratio        : n_circ / (n_mirna + 1e-9)

TOTAL: 22 features. The remaining ~8 can be additional graph metrics you compute
(e.g., assortativity, global efficiency, small-world coefficient, etc.)

POST-PROCESSING:
  → Z-score normalise all numeric features across the full patient cohort
  → Join metadata columns: cancer_type, stage, age, sex, dataset

OUTPUT: features/feature_matrix.csv (rows=patients, cols=features+metadata)

BINARY CLASSIFICATION (main model):
  XGBoost params (starting point, Optuna will tune):

    subsample: 0.6-1.0, colsample_bytree: 0.5-1.0, min_child_weight: 3-10

      Predict on outer test set → store out-of-fold predictions

  METRICS (compute from out-of-fold predictions):

    - sensitivity at 95% specificity (clinical standard)
    - per-stage AUC: for each stage in [I, II, III, IV], compute AUC if >=10 samples

  → Report both in cv_results.csv for Table 2 of the paper

==========================================================
COMMON PITFALLS TO AVOID
==========================================================

1. GEO GSM tables often empty → ALWAYS implement supplementary file fallback
2. Patient IDs across circRNA and miRNA datasets often DON'T overlap (different GEO accessions)
   → Handle this: build circRNA-only patient graphs using population-mean miRNA
3. NetworkX diameter fails on disconnected graphs → ALWAYS work on LCC
4. Betweenness centrality is O(n³) → use k=200 approximation for n>500
5. SHAP beeswarm needs shap.Explanation object, not raw array → wrap it properly
6. gudhi SimplexTree: node IDs must be integers → map string node names to int indices
7. Optuna: suppress verbose output in inner loop (optuna.logging.set_verbosity(WARNING))
8. Z-score normalisation: do AFTER CV split in production; here we do it globally for speed
   → Add a comment noting this limitation
9. miRTarBase URL may change → log the URL being tried and give a manual fallback instruction
10. CircInteractome API can time out → always catch timeout and create placeholder file
